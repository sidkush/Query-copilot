# AskDB — Design Context

## Design Context

### Users
- **Primary**: business analysts, data analysts, PMs, SRE/ops, execs. Same audience as Tableau / Power BI / Looker buyers.
- **Contexts**:
  - Analyst workbench — long multi-hour sessions, dense data, keyboard-first, dark room
  - Exec briefing — 2-min skim on phone / big screen, print-to-PDF, morning coffee
  - LiveOps monitoring — wall-mounted NOC, glance-readable, always dark
  - Story mode — editorial narrative, light cream background, serif body, read like a report
  - Pitch mode — full-screen fullscreen presentation to customers / investors
  - Tableau Classic — power user sheet+dashboard workflow for migrators
- **Jobs to be done**: explore data via NL + SQL, build + share dashboards, present findings, monitor KPIs, autonomous agent workflows.

### Brand Personality
Three words: **precise · authoritative · premium.**

- Voice: confident, terse, no marketing fluff. "The vantage point your data deserves."
- Competes with: Tableau (density + analyst workflow), Power BI (corporate), Looker (semantic rigor), Bloomberg Terminal (ops density).
- Emotional goals: trust (enterprise buyer), focus (no decoration noise), mastery (power user shortcuts), clarity (exec comprehension in seconds).
- Investor-demo stage — every surface must feel intentional and polished. No throwaway screens.

### Aesthetic Direction
**Per-archetype themes** driven by `ARCHETYPE_THEMES` in `frontend/src/components/dashboard/tokens.js`. Each mode has distinct palette, typography, density, chrome:

| Mode | BG | Accent family | Type | Density | Mood |
|------|------|---------------|------|---------|------|
| Briefing | near-black w/ warm tint | gold/amber | serif display + sans body | generous 32/20 | editorial, calm authority |
| Workbench | `#08080d` | cyan/blue functional | JetBrains Mono data | 10px gap, tight | dense, IDE-adjacent |
| LiveOps | `#050508` | traffic-light (green/amber/red) | mono everywhere | 8px gap, tight | NOC terminal, always dark |
| Story | `#FDFBF7` cream | editorial black | serif body | 140px chapter rail | longform magazine |
| Pitch | pure black | brand color per slide | display + sans | full-bleed | theatrical, full-screen |
| Tableau Classic | muted charcoal | Tableau blue | sans + sheet chrome | sheet grid | familiar to migrators |

**Global defaults (dark app chrome):** near-black `#06060e` surfaces, warm-tinted neutrals, OKLCH color space, `--space-*` 4pt scale, no pure black / pure white.

**Anti-references:** Material Design template (too generic), Notion (too doc-like), ChatGPT (too chat-only), any glassmorphism / gradient-text / neon-cyan dashboard.

### Design Principles

1. **Density with rhythm.** Tableau-class density, but with varied spacing — tight groupings, generous separation between sections. No uniform paddings.
2. **Archetype is identity.** Each layout mode must be instantly distinguishable — different type, bg, gap, chrome. Not a color swap.
3. **Data-first chrome.** Chrome (headers, filters, status bars) shrinks when content grows. Never decorate empty space with gradients or glows.
4. **Keyboard-parity everywhere.** Every primary action has a visible shortcut. Cmd+K palette is first-class, not hidden.
5. **No AI slop tells.** Ban list enforced: border-left accent stripes, gradient text, glass cards, neon cyan on dark, centered hero metrics, rounded-icon-above-heading pattern, bounce easing.
6. **Motion serves state, not decoration.** Entrance stagger on first paint OK; no idle shimmer, no bounce, no parallax.
7. **Print-aware.** Briefing + Story must export cleanly to PDF — `@media print` overrides, no glass, no shadows.

### Constraints
- React 19, Vite, Zustand, Tailwind 4, Vega-Lite only chart lib (ECharts fully removed Phase 4c)
- Dark/light theme via CSS vars; archetype overrides resolved-theme where needed (LiveOps force dark, Story force light)
- WCAG AA minimum; AAA where text < 18pt on dark
- Zero new dependencies for polish passes
- Performance: 60fps tile render, Arrow/DataFusion pipeline, WebGL pool for wow charts
