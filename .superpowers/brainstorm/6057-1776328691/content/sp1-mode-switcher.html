  • Left pill: Archetype (Briefing / Workbench / LiveOps / Story / Pitch / Tableau) — changes the layout

  • Main header: Archetype pill only — clean, one concern

C
Smart Auto-Switch Archetype drives edit mode, with manual override
  • Archetype pill is the only switcher. Edit mode is a small badge/indicator, not a pill group.
  • Auto-mapping: Workbench → Pro, Briefing → Default, Pitch → Stage, LiveOps → Default, Story → Default, Tableau → Pro
  • Context-aware: Click a tile → auto-enters edit mode. Close editor → returns to view mode.
  • Manual override: Click the badge to change edit mode if auto-selection is wrong.
  • Pro: Simplest header. Feels intelligent. Less cognitive load.
  • Con: Magic can surprise users. "Why did my mode change?" Override click target is small.
My Recommendation

Option C (Smart Auto-Switch) — the archetype IS the primary decision. Most users won't manually toggle edit modes. Workbench users want Pro editing. Pitch users want Stage. Let the system be smart and get out of the way. The override badge is there for the 5% who want manual control. Keeps the header clean, reduces cognitive load, and matches the "one architecture, three faces" philosophy from wireframe 1.
